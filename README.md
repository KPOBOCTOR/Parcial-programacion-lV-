¿Qué valor agregado tiene el uso de WebComponents en su proyecto?

El uso de WebComponents permitió encapsular la estructura, la lógica y el comportamiento del formulario dentro de un componente personalizado. Esto mejora la organización del código y facilita su reutilización en otras páginas sin necesidad de duplicar funciones o estructura. Además, permite mantener el proyecto más ordenado, modular y escalable, aplicando buenas prácticas modernas del desarrollo FrontEnd.

¿De qué forma manipularon los datos sin recargar la página?

Los datos se manipularon utilizando el evento submit del formulario junto con el método preventDefault(), el cual evita que la página se recargue al enviar el formulario. Posteriormente, se capturaron los valores ingresados por el usuario mediante JavaScript y se actualizó dinámicamente el contenido en pantalla utilizando modificación del DOM, mostrando el resultado inmediatamente sin necesidad de recargar la página.

¿De qué forma validaron las entradas de datos? Expliquen brevemente.

La validación se realizó en dos niveles. Primero, se utilizó validación en HTML mediante el atributo required en los campos obligatorios. Segundo, se implementó validación en JavaScript verificando que los campos no estuvieran vacíos y que los valores ingresados fueran mayores a cero antes de realizar el cálculo. En caso de error, se muestra un mensaje en pantalla sin ejecutar el proceso.

¿Cómo manejaría la escalabilidad futura en su página?

Para manejar la escalabilidad futura del proyecto se podría dividir el código en módulos independientes, crear nuevos componentes reutilizables, implementar almacenamiento de datos con LocalStorage o una base de datos, y conectar la aplicación con un backend para permitir registro de usuarios o almacenamiento de historial. Si el sistema creciera en complejidad, podría migrarse a un framework moderno como React o Vue para mejorar la estructura y mantenimiento del proyecto.
